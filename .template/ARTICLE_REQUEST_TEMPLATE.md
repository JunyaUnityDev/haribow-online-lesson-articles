# 記事制作指示書テンプレート

このテンプレートは ChatClaude が JUNYA とのミーティング後に使う。
JUNYA が Claude Code に投げる用。

`{}` で囲まれた部分を埋めて完成させ、Claude Code に送る。

---

# 記事制作依頼: {記事タイトル}

## 基本情報

- **タイトル**: {タイトル}
- **カテゴリ**: {mindset / technique}
- **スラッグ**: {slug}（URL用、英小文字+ハイフン）
- **想定公開日**: {YYYY-MM-DD}
- **概要(2行)**: {概要}

## 配置先

- ファイルパス: `_drafts/{slug}.md`

## Front Matter（そのままコピー）

```yaml
---
layout: article
title: "{記事タイトル}"
description: "{概要}"
date: {YYYY-MM-DD}
category: {mindset または technique}
slug: {slug}
---
```

## 記事構成

### 読者層

{読者層、前提知識、想定する状況}

### 結論

{この記事で伝えたい一番大事なこと}

### 構成

1. {見出し1}
   - 内容のメモ
2. {見出し2}
   - 内容のメモ
3. ...

### 文体・トーン

- {例: 既存記事と同じ、淡々と論理的、ですます調}
- {例: 感情論を避け、構造で説明}
- 禁止: 鼓舞・煽り・「絶対」「必ず」のような断定

### 参考資料・リンク

- {URL や参考にしたい資料}

### 動画・画像・音源

- {動画埋め込みする場合の VIDEO_ID と挿入位置}
- {画像 URL（既に Drive 等にある）と挿入位置}
- {音源埋め込みする場合のファイル名と挿入位置}

## 重要な制約

- 既存4記事（`_posts/` 配下）のトーン・構造と整合性を保つ
- 結論を最初に出す（既存記事のスタイル）
- 章番号は `## 第N章｜題名` 形式（mindset 系記事の場合）
- 子供から大人まで読める語彙（漢字・専門用語の比率に注意）

## 実装後の作業

1. `_drafts/{slug}.md` に md ファイル作成
2. JUNYA に確認依頼
3. JUNYA が確認・微調整
4. （公開時）JUNYA が `_posts/{YYYY-MM-DD}-{slug}.md` にリネーム移動

## 報告フォーマット

```markdown
## 記事制作完了報告

### 作成したファイル
- {ファイルパス}

### 文字数: {N文字}

### 章立て
- 第1章: ...
- 第2章: ...

### JUNYA への確認依頼
- 内容のチェック
- 文体・トーンの違和感確認
- メディア（画像・動画・音源）の挿入位置確認
```
